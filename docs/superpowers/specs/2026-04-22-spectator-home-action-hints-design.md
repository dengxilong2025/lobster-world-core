# spectator/home 行动建议增强（v0.3-M1+）Design

**目标**：在不改 API 的前提下，增强 `/api/v0/spectator/home` 返回的 `world.summary` 中“建议：”的可执行性，让玩家看完能直接写出可触发的 intent，并预期会出现哪些事件（尤其是 v0.3-M1 新增的 `trade_agreement / alliance_formed / treaty_signed`）。

**范围**：仅调整 `internal/gateway/world_summary.go` 的建议生成逻辑与对应测试；不改变 JSON 结构（仍为 `stage + summary[] + state{...}`）。

**非目标**：
- 不新增字段（不做 `action_hints[]` 结构化扩展）
- 不做复杂 AI 推理或跨事件链检索（只用当前 state + 最近 narrative 字符串）
- 不更改 stage 判定优先级

---

## 1) 现状

`deriveWorldSummary(st, recent)` 当前输出：
- `阶段：...`
- `近期：...`
- `看点：...`
- `风险：...`（若满足阈值）
- `建议：提交一个“…”意图...`（单条建议，偏泛）

问题：
- 建议虽“可操作”，但不够具体：缺少可复制的关键词集合、缺少“预期事件类型”（尤其是外交/贸易剧本已落地但建议未利用）
- 当世界处于多风险（饥荒+低信任+失序）时，玩家不知道先做什么

---

## 2) 设计原则（防偏离）

1) **不改 API**：输出仍为 `summary: []string`，最多只改变其内容与条目数量。
2) **决定论**：只依赖输入 `st` 与 `recent`；不引入随机与时间。
3) **建议必须“可复制粘贴”**：每条建议都包含 2~4 个关键词，玩家随便选一个写进 goal 就能触发预期事件/效果。
4) **建议必须“可验证”**：每条建议说明“预期会看到的事件类型”（用于回放与验收）。

---

## 3) 输出结构（保持兼容）

在 `ws.Summary` 中允许出现 **2 条**以 `建议：` 开头的行（主建议 + 次建议）：

- 主建议：依据“最高优先级风险”生成（优先解决最致命指标）
- 次建议：依据 stage + recent 生成（用于推动叙事/利用新剧本）

两条建议都必须满足：
- 包含 `提交` 字样（保持现有测试假设）
- 不与 `看点：` 文案完全重复（保持现有去重/防重复逻辑）

---

## 4) 建议生成规则（v0）

### 4.1 风险优先级（用于主建议）

按以下优先级选“主建议”：
1) `Food <= 20`（饥荒风险）
2) `Conflict >= 60`（战乱风险）
3) `Trust <= 25`（低信任风险）
4) `Order <= 20`（失序风险）
5) 默认（萌芽/启蒙/扩张的推进型建议）

### 4.2 建议模板与关键词（必须包含）

> 约束：建议必须包含关键词与预期事件类型（至少包含一种）。

**饥荒（Food<=20）主建议：贸易优先**
- 关键词：`贸易 / 集市 / 交换 / 商路`
- 预期事件：`trade_agreement`
- 文案模板（示例）：
  - `建议（优先补给）：提交“贸易/集市/交换/商路”意图，观察是否触发 trade_agreement 并提升 food/trust`

**战乱（Conflict>=60）主建议：外交降冲突**
- 关键词：`停战 / 谈判 / 条约 / 结盟`
- 预期事件：`treaty_signed` 或 `alliance_formed`
- 文案模板（示例）：
  - `建议（优先降冲突）：提交“停战/谈判/条约/结盟”意图，观察 treaty_signed / alliance_formed 是否出现并降低 conflict`

**低信任（Trust<=25）主建议：结盟修复关系**
- 关键词：`结盟 / 联盟 / 合作 / 互助`
- 预期事件：`alliance_formed`（优先）/ `trade_agreement`（备选）

**失序（Order<=20）主建议：整顿秩序**
- 关键词：`整顿 / 裁决 / 执法 / 议会`
- 预期事件：不强制（仍走 v0 intent delta），但要明确“观察 order 是否回升”

**默认（推进型）主建议：探索/贸易/合作**
- 关键词：`探索 / 贸易 / 合作`
- 预期事件：`trade_agreement`（若命中贸易关键词）或普通 delta

### 4.3 次建议（stage/recent 引导）

次建议用于“叙事推进与验证新剧本”，规则：
- 若 `stage == 战乱` 或 recent 含 `背叛/翻脸`：次建议偏外交（结盟/谈判）
- 若 `stage == 饥荒`：次建议偏补给（狩猎/补给）与贸易二选一（避免与主建议完全重复）
- 若 `stage == 启蒙`：次建议偏研究/制度（现有路径）
- 若 `stage == 扩张`：次建议偏外交/迁徙（现有路径）

## 5) 测试门禁（TDD）

在 `internal/gateway/world_summary_test.go` 新增/加强测试：

1) **饥荒场景**：必须至少有一条 `建议：` 同时包含：
   - 关键词之一：`贸易`/`集市`/`交换`/`商路`
   - 预期事件字符串：`trade_agreement`
2) **战乱场景**：必须至少有一条 `建议：` 同时包含：
   - 关键词之一：`停战`/`谈判`/`条约`/`结盟`
   - 预期事件字符串：`alliance_formed` 或 `treaty_signed`
3) 继续保留原门禁：`建议：` 存在、包含 `提交`、不与 `看点：` 重复
