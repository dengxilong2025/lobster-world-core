# /ui/assets 导出留档清单（本地持久化）设计说明

## 目标

在 `/ui/assets` 中，为“导出 3×3 PNG（tiles.base）”增加**留档清单**能力：

1) 每次导出成功后自动记录一条条目（时间戳 / 分类 / 资源路径 / 导出文件名）
2) 留档在浏览器本地持久化（localStorage），刷新页面不丢
3) 提供 3 个操作：
   - 复制留档（复制为文本，便于粘贴到群/文档）
   - 下载留档 JSON（便于长期存档/后续处理）
   - 清空留档

## 非目标
- 不做服务端存储与同步
- 不做批量导出（后续可再加）

## 交互
- 在 `/ui/assets` 页面顶部（header 下方）增加一个“导出留档”面板，默认展开：
  - 显示最近 N 条（例如 200 条上限，超出自动裁剪）
  - 3 个按钮：复制 / 下载 JSON / 清空

## 数据结构

localStorage key：`lw_assets_export_log_v1`

条目结构（JSON）：
```json
{
  "ts": "20260420-142233",
  "cat": "tiles.base",
  "relPath": "tiles/base/tile_grass_basic_01_w256_h256.png",
  "filename": "tile_grass_basic_01_w256_h256__3x3__20260420-142233.png"
}
```

## 验收方式
- 手动：
  1) 打开 `/ui/assets` → tiles.base → 打开任意瓦片弹窗 → 导出 3×3 PNG
  2) 回到页面上方“导出留档”面板，能看到新条目增加
  3) 刷新页面，条目仍在
  4) “复制留档”能复制出多行文本；“下载 JSON”能下载文件；“清空”能清空
