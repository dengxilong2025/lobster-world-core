# 资源包盘点报告（3/7）— P0_assets.zip

> 本包特点：目录结构清晰（`ui/` + `tiles/`），但存在两个明显问题：  
> 1) **大量 UI 资源用“大画布”输出**（实际尺寸 1024×1024 / 1344×576 / 720×1280），与文件名 `_w.._h..` 不一致  
> 2) 图片虽然是 RGBA，但 **alpha 全为 255（等同于没有透明）**，UI 需要先去底转透明  
> 另外：部分 tiles 顶部出现“竖向拉丝/渲染断层”伪影，建议作为探索素材保留，不建议当 P0 地表瓦片。

## 2) 可用性分级

### ✅ UI（frames/badges/icons）：可用，但必须走“我这边修复流程”
原因：
- 需要去底转透明
- 需要按 `_w.._h..` 做缩放到目标尺寸（32/64/128/256 等）

### ⚠️ tiles：不建议作为 P0 Type A 地表无缝瓦片
观察到的风险：
- 部分 tile 顶部有明显“竖向拉丝/断层”伪影（AI 渲染常见），拼图时会非常显眼
- 画面包含较多大主体（石头/草丛/边缘灌木），更像“场景块”而非“基础无缝纹理瓦片”

建议：
- 先作为探索库保留（可用于灵感/占位场景块）
- P0 的 Type A（grass/dirt/sand/rock）仍按手册 9.6 重新做

我对本包做了“UI 专用”自动修复：
