## 0. 你要做的事（先看这一段）

P1 我们优先要你补齐三类东西：

3) **一组“地表过渡瓦片（Type C）”**：草→土、土→沙、土→石、沙→石（P1 选做；如果时间够强烈建议做）

> 你可以像今天一样，明天输出多个版本的压缩包；我会按本规范验收并挑选最合格的入库。

## 1. 总体风格要求（Stone Age / 部落手绘 / 暖色）

### 1.1 关键词（你每次生成都要对齐）
- 石器时代、部落感、羊皮纸/土黄色调、手绘绘本、暖色、自然材质（木/骨/石/皮革）
- 颗粒感/纸张纹理可以有，但不要“脏到看不清”
- 光照：柔和暖光，避免冷色霓虹

### 1.2 风格红线（违反就不采用）
- **任何文字、数字、字母、水印、签名、Logo**（包括很小的 “hi.”、英文字母 “A” 这种）
- 赛博/霓虹/冷色主导
- 过强的写实照片贴图（会破坏整体手绘一致性）

---

## 2. 统一规格（尺寸/格式/透明/命名）

### 2.1 文件格式
- **PNG**（推荐 PNG-24 / RGBA）
- 色彩空间：sRGB

### 2.2 尺寸（必须严格一致）
- **基础地表瓦片（Type A / Type C）**：`256×256`
- **地图贴片/道具/建筑（props）**：`256×256`（如果你要做）
- **UI icons**：两套都要  
  - `128×128`（主用）  
  - `64×64`（缩略/列表用）

### 2.3 透明通道（alpha）
- 地表瓦片（Type A / Type C）：允许不透明（alpha 全 255 也可以）
- UI icons：必须透明底（RGBA），且 **背景必须完全透明**
- 如果你不会抠图：**可以交付白底原图**，但请同时交付“未抠图版本”（见 7.2），我这边更容易做高质量透明

### 2.4 命名规则（我会用脚本验收）

#### 2.4.1 基础地表（Type A）
```
tile_{biome}_basic_{variant}_w256_h256.png
```
例：
- `tile_grass_basic_01_w256_h256.png`
- `tile_dirt_basic_01_w256_h256.png`

#### 2.4.2 过渡瓦片（Type C）
```
tile_{from}_to_{to}_transition_{variant}_w256_h256.png
```
例：
- `tile_grass_to_dirt_transition_01_w256_h256.png`

#### 2.4.3 UI icons
```
ui_icon_{name}_w{size}_h{size}.png
```
例：
- `ui_icon_event_w128_h128.png`
- `ui_icon_event_w64_h64.png`

## 3. P1 必做：基础地表瓦片（Type A，无缝平铺）

> 这是 P1 最重要的交付。  
> 目标是：地图“铺起来”时看起来像连续地表，而不是一块块插画拼图。

### 3.1 必做清单（最小集合）
每个地表至少做 **3 个变体**（01/02/03），避免重复感：

1) 草地：`tile_grass_basic_01..03_w256_h256.png`
2) 泥土：`tile_dirt_basic_01..03_w256_h256.png`
3) 沙地：`tile_sand_basic_01..03_w256_h256.png`
4) 岩地/碎石地：`tile_rock_basic_01..03_w256_h256.png`

### 3.2 建议做清单（如果你有时间）
5) 雪地：`tile_snow_basic_01..03_w256_h256.png`  
6) 浅水纹理：`tile_water_shallow_01..03_w256_h256.png`

### 3.3 Type A 的“内容尺度”要求（非常重要）
Type A 只能是“纹理”，不要出现大主体：
- ✅ 可以有：细小砂砾、微小草叶点、裂纹、泥点、细碎石子
- ❌ 不要有：大石头、明显灌木边缘、木桩、骨头、建筑残骸、路径边界

原因：大主体会导致“重复图案”极其明显，一铺图就穿帮。

## 4. P1 选做但强烈建议：地表过渡瓦片（Type C）

> 如果 Type A 做得很好，下一步地图质感提升最大的就是“过渡边缘自然”。

### 4.1 过渡对（建议优先级）
1) 草 → 土：`tile_grass_to_dirt_transition_01..02_w256_h256.png`
2) 土 → 沙：`tile_dirt_to_sand_transition_01..02_w256_h256.png`
3) 土 → 石：`tile_dirt_to_rock_transition_01..02_w256_h256.png`
4) 沙 → 石：`tile_sand_to_rock_transition_01..02_w256_h256.png`

### 4.2 过渡瓦片的样式建议（不要求你一次做完美）
- 过渡边缘“软”，像自然侵蚀/泥土混沙/草根稀疏，而不是硬切线
- 不要做成“边框贴图”（边缘像贴上去的带子）

## 5. P1 必做：2 个 UI 图标（无字）

### 5.1 需要新增制作的图标
1) `event`（事件）：`ui_icon_event_w128_h128.png` + `ui_icon_event_w64_h64.png`
2) `betrayal`（背叛/反水）：`ui_icon_betrayal_w128_h128.png` + `ui_icon_betrayal_w64_h64.png`

### 5.2 图标设计方向（你可以按任意方案出多个版本）
**event** 可选方向（任选其一）：
- 小篝火 + 火花
- 鼓/号角（发布号令）
- 卷轴/石板（公告）

**betrayal** 可选方向（任选其一）：
- 断裂的骨签/断裂的盟约绳结
- 两只手相背 / 破碎的图腾
- 破裂面具（象征背叛）

### 5.3 图标制作要点
- 画面主体尽量填满 70%~85% 画布（否则 64×64 会太小）
- 边缘要干净（避免黑边/白边晕染）
- **绝对禁止任何字母/文字**（包括小小的签名）

## 6. 验收标准（我会这样判定“合格/不合格”）

### 6.1 自动验收（硬指标）
- 尺寸严格匹配文件名（例如 `_w256_h256` 就必须真的是 256×256）
- PNG 可打开，无损坏
- UI icon 必须带 alpha，且背景透明（不是米白底）
- 文件名全小写 + 下划线 + 按规则命名

### 6.2 人眼验收（软指标，但会强影响采用）

#### Type A（无缝平铺）必须过关
我会做 3×3 拼贴检查：
- 横向/纵向拼起来 **看不到硬接缝**
- 不能出现明显“中心重复印章感”（像墙纸）

#### 风格一致性
- 是否整体偏暖、偏手绘
- 是否出现明显 AI 伪影（竖向拉丝、结构断裂、怪异霓虹色块）

## 7. 制作方法建议（给你一个稳的工作流）

### 7.1 生成前：把提示词写成“不可违背的约束”
建议你把提示词分成三段：
1) 主题/风格（部落手绘、石器时代、暖色）
2) 规格约束（256×256、seamless tileable、no text）
3) 禁止项（no watermark, no signature, no letters）

示例（给 Type A 草地）：
- “stone age tribal hand-painted ground texture, warm earthy palette, seamless tileable 256x256, subtle grass blades and dirt speckles, no large objects, no borders, no text, no watermark, no signature”

### 7.2 强烈建议：同时保留“未抠图原图”
你每张图尽量保存两份（打包时可放一个 `raw/` 一个 `export/`）：
- `raw/`：白底/原图（不抠图、不去底）
- `export/`：透明底 PNG（如果你会做）

原因：如果透明处理把“白色高光/亮部”误扣掉，我可以用 raw 版本补回颜色，质量会明显更好。

### 7.3 无缝瓦片自检（你交付前自己先做一遍）
最简单方式：
1) 把同一张瓦片复制成 3×3（任意软件都行）
2) 放大看接缝：是否出现竖线/横线
3) 退远看重复：是否出现明显重复图案

如果你会 PS/Photopea：
- 用 Offset（水平/垂直各偏移一半）把接缝移到中心，再修中心十字
