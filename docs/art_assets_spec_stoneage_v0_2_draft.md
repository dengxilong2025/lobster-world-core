# v0.2 美术资源规范（石器时代·楼兰手绘等轴）— 草案 v0（给小文的“啰嗦版”）

> 这份文档的目标：**让第一次做游戏美术/素材制作的人，也能按步骤做对**。  
> 你只需要照着“✅必须做到 / ❌禁止出现 / 📦交付清单 / 📐分辨率表 / 🗂目录与命名”来做。  
> **最重要的铁律：所有 UI/元素图片里都不要出现任何文字/数字/水印/Logo**（包括 AI 生成的乱码）。

## 0. 我们到底要做什么（给新手的解释）

我们现在处在“**先做出可玩的壳（Web 观战/回放）**”阶段，美术资源的用途主要是：

1) **让页面看起来像游戏**（风格统一、质感到位）  
2) **不阻碍工程开发**（资源可替换、可扩展、命名清楚、不带文字）  
3) 后续还能升级到更像“正式游戏”（可能接入 3D/动画/音效）

因此：第一批美术优先做 **“可替换的 UI 组件 + 图标 + 少量场景/地图/角色”**。  
不要一开始就追求把整套 HUD/菜单做成一张大图（那会导致返工巨大）。

## 1) 主题与风格（石器时代 / 楼兰手绘）— **硬规则**

### 1.1 主题关键词（必须）
- 石器时代 / 原始部落 / 篝火 / 兽皮 / 骨饰 / 石斧木矛 / 图腾柱 / 岩画 / 羊皮纸 / 烟熏痕迹
- 氛围：温暖、粗粝、朴素、手作感（不是精致奢华）

### 1.2 美术风格（必须）
- **手绘绘本质感**：笔触、轻微颗粒、柔和边缘
- **等轴视角 (Isometric)**：建筑/地形/地图类尽量保持等轴 30° 规则
- 线条：深褐色线条为主（不是纯黑线），线条不需要完全均匀（要有手绘感）

### 1.3 色彩（你给的主色盘 + 我补的中性色）

**主色（必须占主导）**
- 土黄：`#C4A35A`
- 深褐：`#4A3728`
- 篝火橙：`#E67E22`

**辅助（少量点缀）**
- 枯黄：`#D4A84B`（丰收/秋叶）
- 沙色：`#E8D4A8`（羊皮纸/沙漠）
- 草绿：`#6B8E23`（植物点缀，不能满屏绿）
- 夜空蓝：`#1A1A2E`（夜景少量使用）
- 血迹红：`#8B0000`（战斗点缀，少量使用）

### 1.4 风格红线（绝对禁止）
> 只要出现其中任何一条，这张图就是“废稿/只能放探索区”，不能进入生产资源。

❌ 禁止出现：
- 写实摄影感
- 赛博朋克/霓虹发光/高饱和荧光色
- 日式动漫（大眼、二次元脸）
- 欧美 3D 渲染质感（像 3A 游戏截图）
- 冷色调为主（蓝紫霓虹）
- **任何文字/数字/水印/Logo/签名/英文单词/乱码字符**

✅ 必须保持：
- 暖色大地色系为主
- 手绘插画/绘本质感
- 原始部落感
- 画面“能读懂”（不靠文字解释）

## 2) **“禁止文字”铁律（非常重要，单独强调）**

### 2.1 为什么我们强制不带文字？
因为游戏里的文字要由程序渲染（多语言/可改/可适配）。  
如果你把文字画进 PNG：
- 程序无法翻译
- 缩放会糊
- 以后改词要重画整张图
- AI 还经常生成乱码（非常影响质量）

### 2.2 具体禁止范围（新手常踩坑）
以下全部算“文字”，一律不允许出现在图片里：
- FOOD / WOOD / STONE / RESOURCE 这类英文
- TRIBE ON KINGDOM 这类标题
- 任何按钮上的字（PLAY/START/SETTING）
- 地图上的地名标注（哪怕很小）
- 角色立绘角落的水印、作者签名、版权字样
- AI 生成的假字母、假符号（看起来像字就不行）

### 2.3 正确做法（替代方案）
✅ 用“图形”表达：  
- 用图标代表资源（木材/石头/食物）
- 用颜色/徽章代表状态（风险/冲击/背叛）
- 留空白区域，给程序放文字（比如按钮中间留白）

## 3) 资源库与生产资源的区别（避免返工的关键）

我们把资源分成两类：

1) **探索库（Exploration / Library）**：允许“尺寸乱一点/构图试验/内容未拆分”  
   - 目的：找风格、找感觉、做 moodboard
2) **生产库（Production / Game Assets）**：必须严格按规格  
   - 目的：直接被程序引用（稳定、可维护）

> 你目前上传的 `library/` 属于探索库，很正常；我们会在此基础上整理出生产规范。

## 9.5) 【教程】地形瓦片（Tile）到底怎么做（给不会 PS 的小文也能照做）

> 先说一个容易误会的点：你现在 `scenes/terrain/` 里的很多图更像“风景插画”，不是可拼接的 **瓦片**。  
> **瓦片**的目标是：程序把它复制很多次铺成地图时，**边缘完全对得上、不会出现明显接缝**。

### 9.5.1 我们先把瓦片分成两类（降低难度）

**Type A：纯纹理瓦片（P0 必做，最简单）**  
- 只包含：草地/沙地/岩地/雪地/泥地这种“地表纹理”
- 不包含：明显主体（比如一棵大树在正中间）
- 优点：最容易做到“无缝平铺”

**Type B：带地物装饰瓦片（P1 再做，难度更高）**  
- 在纹理上加少量：石头、枯草、骨片、脚印、裂纹、图腾碎片等
- 优点：更像游戏地图
- 难点：更容易在边缘露馅、重复感强

> 结论：**P0 只做 Type A**，等我们铺图跑通后再做 Type B。

### 9.5.2 生产尺寸先定死（先 Web 能用）

**P0 生产推荐：256×256 PNG（无字）**  
探索阶段如果小文习惯大画布，也可以先画 1024×1024，然后再缩小导出到 256×256。

为什么 256×256？
- 是游戏行业常用瓦片尺寸（易压缩、易打图集、好管理）
- Web 和引擎都好用

### 9.5.3 “无缝平铺”怎么保证？（最关键）

你只要记住一句口诀：  
> **把接缝移到中间修掉，再移回去。**

#### 方法 A（推荐）：用“偏移 / Offset”功能（PS / Photopea / Krita 都有类似）
1. 新建画布：256×256（或 512×512），先画出一张草地/沙地纹理  
2. 执行偏移（Offset）：  
   - 水平偏移：宽度的一半（256 的一半是 128）  
   - 垂直偏移：高度的一半（同样是 128）  
3. 偏移后：你会看到“原本在边缘的接缝”跑到画面中间形成一条十字  
4. 用修补/涂抹/复制纹理，把中间的十字接缝修到看不出来  
5. 再偏移回去（同样偏移 128/128），导出即可

✅ 自检方法：  
把这张瓦片复制 3×3 拼起来，观察：  
- 横向接缝处是否有“断裂线/颜色突变/纹理突然变化”  
- 纵向接缝同理  
如果看不出接缝，就合格。

#### 方法 B（不会 Offset 也能做）：用“九宫格拼贴自检”
1. 先画一张瓦片  
2. 把它复制成 3×3（随便用画图工具粘贴也行）  
3. 直接观察拼接线  
4. 如果出现明显接缝：回到原图，在边缘区域“做过渡”（把边缘的明显纹理打散）

> 方法 B 不如 Offset 稳，但给完全不会 PS 的小文也能先凑出可用版本。

### 9.5.4 瓦片常见失败原因（避免踩坑）

❌ 失败原因 1：中心放了一个“唯一主体”  
- 比如每块草地中间都有一朵花  
- 平铺后会出现“整齐重复的花阵”，很假  
✅ 做法：把“强特征”拆成单独的装饰物瓦片（Type B），不要放在 Type A 里

❌ 失败原因 2：边缘有明显斜线/石头贴边  
- 平铺会出现断裂  
✅ 做法：边缘尽量“干净”，装饰物尽量离边缘留出 10% 安全区

❌ 失败原因 3：颜色对比过强  
- 平铺的格子感更明显  
✅ 做法：Type A 纹理对比要温和，靠细节与颗粒感丰富，而不是靠强边缘

## 9.6) 【规范】瓦片地图资源要求（Tilemap / Tiles）— 小文执行版（超细）

> 结论先说：  
> **小文交付给程序的瓦片，一律先按“单瓦片独立 PNG 文件”**（最好迭代、最好验收）。  
> 后续如果需要性能/引擎友好，我会再把这些单文件 **自动打包成 Tileset/Atlas（图集）**。

### 9.6.1 绝对铁律（违反就判不合格）
1) **瓦片图片里不能出现任何文字/数字/水印/Logo/签名/乱码字母**  
2) **同一个瓦片集合必须统一尺寸**（不要一会 1024×1024，一会 1248×832）  
3) **瓦片必须可平铺（tileable / seamless）**：复制成 3×3 拼贴时看不出接缝  
4) **风格红线**：不得出现写实/霓虹/赛博朋克/冷色为主/日漫脸（石器时代暖色手绘）

### 9.6.2 我们当前选择的瓦片体系（v0.2/v0.3 推荐）
#### P0/P1：先用“方形瓦片纹理”（最稳）
- **瓦片尺寸（生产）**：`256×256`
- **格式**：PNG-24，sRGB
- **透明**：建议不透明（纯地表纹理），也可以透明（两种都行，但要统一）
- **用途**：先把地图“铺得起来”，支持快速可玩雏形

#### P2 以后（可选升级）：等轴菱形瓦片
- 例如菱形 `256×128`（2:1）+ 统一锚点
- 这套对程序与美术要求更高，我们等玩法节奏稳定后再做，不作为 P0 必做

### 9.6.3 瓦片分类（按难度从低到高）
#### Type A（P0 必做）：纯地表“纹理瓦片”
只包含草/土/沙/石/雪/浅水等“地表纹理”，**不放大主体**。

合格标准：
- 3×3 拼贴无明显接缝
- 不出现明显“中心主体重复”（例如每块中央都有一朵花）

#### Type B（P1 再做）：带装饰地物瓦片（Overlay/Detail）
在 Type A 基础上叠加少量装饰：石头、枯草、骨片、脚印、裂纹、图腾碎片。

合格标准：
- 装饰物不要贴边（推荐离边缘至少 10% 安全区）
- 3×3 拼贴不会出现断裂/切断
- 重复感可接受（建议同一类装饰至少 3 个变体）

#### Type C（P2 再做）：地形过渡瓦片（Transition）
用于“草地→土”“土→沙”“沙→石”等边缘自然过渡。

> 这类最容易难产，不强制小文 P0 做。  
> 我们可以先用程序端简化过渡（或先不用过渡），等后面再补齐。

### 9.6.4 P0 交付清单（最低能铺图）
小文第一批只要交 **Type A**，最少这 6 张（每张都是 256×256）：

1) `tile_grass_basic_01_w256_h256.png`（草地）
2) `tile_dirt_basic_01_w256_h256.png`（泥土）
3) `tile_sand_basic_01_w256_h256.png`（沙地）

> 如果时间紧：先交前 3 张（grass/dirt/sand），也能铺起来。

### 9.6.5 命名规则（必须遵守，方便我自动处理/打图集）
统一格式（全小写 + 下划线）：
```
tile_{biome}_{kind}_{variant}_w256_h256.png
```

- biome：`grass` / `dirt` / `sand` / `rock` / `snow` / `water`
- kind：`basic`（纯纹理）/ `detail`（带装饰）/ `transition`（过渡）
- variant：`01` `02` `03`…

示例：
- `tile_grass_basic_01_w256_h256.png`
- `tile_grass_detail_stones_01_w256_h256.png`
- `tile_grass_to_dirt_transition_01_w256_h256.png`

### 9.6.6 “无缝平铺”自检方法（小文交付前必须做）
#### 最简单自检（不需要 PS）
- 把瓦片复制成 3×3（任何工具都行）
- 检查四条拼接线：
  - 横向接缝有没有“线”
  - 纵向接缝有没有“线”
  - 是否出现明显重复图案（像墙纸一样重复）

#### 进阶自检（会 PS/Photopea 更好）
- 做 Offset 偏移：水平/垂直都偏移一半（256 的一半是 128）
- 接缝会出现在中心十字
- 把中心十字修掉，再偏回去

> 如果小文不会：没关系，先按“最简单自检”做到“不明显接缝”也可以，我们后续可用工具辅助修。

### 9.6.7 风格细节要求（石器时代暖色手绘）
必须具备：
- 暖色土黄/深褐/篝火橙为主（不满屏绿）
- 轻微颗粒、笔触、绘本质感
- 边缘柔和但不要糊成一片

禁止：
- 霓虹/赛博/冷色主导
- 写实照片纹理（太真实会与整体手绘冲突）
- 任何字母/数字/标记
