## 0. 我们到底要做什么（给新手的解释）

我们现在处在“**先做出可玩的壳（Web 观战/回放）**”阶段，美术资源的用途主要是：

1) **让页面看起来像游戏**（风格统一、质感到位）  
2) **不阻碍工程开发**（资源可替换、可扩展、命名清楚、不带文字）  
3) 后续还能升级到更像“正式游戏”（可能接入 3D/动画/音效）

因此：第一批美术优先做 **“可替换的 UI 组件 + 图标 + 少量场景/地图/角色”**。  
不要一开始就追求把整套 HUD/菜单做成一张大图（那会导致返工巨大）。

## 1) 主题与风格（石器时代 / 楼兰手绘）— **硬规则**

### 1.1 主题关键词（必须）
- 石器时代 / 原始部落 / 篝火 / 兽皮 / 骨饰 / 石斧木矛 / 图腾柱 / 岩画 / 羊皮纸 / 烟熏痕迹
- 氛围：温暖、粗粝、朴素、手作感（不是精致奢华）

### 1.2 美术风格（必须）
- **手绘绘本质感**：笔触、轻微颗粒、柔和边缘
- **等轴视角 (Isometric)**：建筑/地形/地图类尽量保持等轴 30° 规则
- 线条：深褐色线条为主（不是纯黑线），线条不需要完全均匀（要有手绘感）

### 1.3 色彩（你给的主色盘 + 我补的中性色）

**主色（必须占主导）**
- 土黄：`#C4A35A`
- 深褐：`#4A3728`
- 篝火橙：`#E67E22`

**辅助（少量点缀）**
- 枯黄：`#D4A84B`（丰收/秋叶）
- 沙色：`#E8D4A8`（羊皮纸/沙漠）
- 草绿：`#6B8E23`（植物点缀，不能满屏绿）
- 夜空蓝：`#1A1A2E`（夜景少量使用）
- 血迹红：`#8B0000`（战斗点缀，少量使用）

### 1.4 风格红线（绝对禁止）
> 只要出现其中任何一条，这张图就是“废稿/只能放探索区”，不能进入生产资源。

❌ 禁止出现：
- 写实摄影感
- 赛博朋克/霓虹发光/高饱和荧光色
- 日式动漫（大眼、二次元脸）
- 欧美 3D 渲染质感（像 3A 游戏截图）
- 冷色调为主（蓝紫霓虹）
- **任何文字/数字/水印/Logo/签名/英文单词/乱码字符**

✅ 必须保持：
- 暖色大地色系为主
- 手绘插画/绘本质感
- 原始部落感
- 画面“能读懂”（不靠文字解释）

## 2) **“禁止文字”铁律（非常重要，单独强调）**

### 2.1 为什么我们强制不带文字？
因为游戏里的文字要由程序渲染（多语言/可改/可适配）。  
如果你把文字画进 PNG：
- 程序无法翻译
- 缩放会糊
- 以后改词要重画整张图
- AI 还经常生成乱码（非常影响质量）

### 2.2 具体禁止范围（新手常踩坑）
以下全部算“文字”，一律不允许出现在图片里：
- FOOD / WOOD / STONE / RESOURCE 这类英文
- TRIBE ON KINGDOM 这类标题
- 任何按钮上的字（PLAY/START/SETTING）
- 地图上的地名标注（哪怕很小）
- 角色立绘角落的水印、作者签名、版权字样
- AI 生成的假字母、假符号（看起来像字就不行）

### 2.3 正确做法（替代方案）
✅ 用“图形”表达：  
- 用图标代表资源（木材/石头/食物）
- 用颜色/徽章代表状态（风险/冲击/背叛）
- 留空白区域，给程序放文字（比如按钮中间留白）

## 3) 资源库与生产资源的区别（避免返工的关键）

我们把资源分成两类：

1) **探索库（Exploration / Library）**：允许“尺寸乱一点/构图试验/内容未拆分”  
   - 目的：找风格、找感觉、做 moodboard
2) **生产库（Production / Game Assets）**：必须严格按规格  
   - 目的：直接被程序引用（稳定、可维护）

> 你目前上传的 `library/` 属于探索库，很正常；我们会在此基础上整理出生产规范。

## 5) 命名规则（小文照抄就行）

### 5.1 基本规则
- 全小写
- 下划线分隔（snake_case）

## 6) 分辨率/格式规范（v0.2 先定“能用 + 不返工”的标准）

### 6.1 总原则
- **优先使用 2 的幂尺寸**：64/128/256/512/1024/2048/4096  
  （显卡/压缩/图集更友好）
- v0.2 Web 原型阶段：允许 1024×1024 作为“统一探索画布”
- 进入生产库前：尽量收敛到“每类资源 1-2 个固定尺寸”

### 6.2 各类型资源规格表（小文主要看这个）

| 类型 | 用途 | ✅生产推荐尺寸 | ✅可接受尺寸（探索） | 格式 | 是否透明 |
|---|---|---:|---:|---|---|
| 场景 exterior/interior/events | 观战/事件背景 | 1024×1024（先统一） | 1024×1024 | PNG | 否 |
| 世界地图 bigmap | 世界概览 | 2048×1024（推荐） | 1280×720 或 1024×1024（探索可用） | PNG | 否 |
| 地形瓦片 terrain（真正的瓦片） | 拼地图 | 256×256（透明，瓦片在中间） | 512×512 | PNG | 是 |
| 角色头像 portrait（单个角色） | 列表/详情 | 512×512（头像或半身） | 1024×1024（探索） | PNG | 是（推荐） |
| 建筑（单体） | 等轴建筑 | 512×512（透明） | 1024×1024（探索） | PNG | 是 |
| 物品/道具 icon | 背包/资源 | 128×128（透明） | 256×256（探索） | PNG | 是 |
| UI 图标（单个） | 按钮/状态 | 64×64（主）+ 128×128（高清） | 128×128 | PNG/SVG | 是 |
| UI 面板/边框切片 | 组件化 UI | 见“切片规则” | 1024×1024 整图（仅探索） | PNG/SVG | 视情况 |

## 7) UI 资源“怎么做才不会返工”（非常啰嗦但很重要）

### 7.1 UI 整图（如主菜单/设置）只当“概念稿”
你们现有 `ui_screen_main_menu.png` 这种整屏图很好看，但它是 **概念稿**：  
✅ 用于确定风格与布局氛围  
❌ 不能直接进入生产（因为里面通常有文字、按钮不可拆）

### 7.2 v0.2 真正需要交付的 UI，是“组件”（不带字）
第一批 UI 组件（P0）请小文按下面做：

#### A) 面板边框（Panel Frame）
目标：做 1 套“石器时代羊皮纸+木骨架/骨饰”的面板框，程序可以复用在任何弹窗。

**交付（九宫格切片）**
- 角：4 张 `32×32`（tl/tr/bl/br）
- 边：4 张（top/bottom 用 `128×32`，left/right 用 `32×128`）
- 中心填充：1 张 `256×256`（可平铺的羊皮纸纹理）

命名示例：
- `ui_frame_panel_main_corner_tl_w32_h32.png`
- `ui_frame_panel_main_edge_top_w128_h32.png`
- `ui_frame_panel_main_fill_w256_h256.png`

#### B) 徽章/标签底（Badge Background）
目标：用于显示“阶段/近期/看点/风险/建议”这些词（注意：底图不带字！）。

## 11)（可选）小文写 Prompt 时的“安全模板”（防止跑偏）

> 目的：减少生成出霓虹/写实/带字的概率。

✅ 推荐模板（中文也行）：
```
手绘绘本插画风格，石器时代原始部落主题，暖色大地色系（#C4A35A #4A3728 #E67E22），
等轴视角/或平视角色立绘，轻微颗粒与羊皮纸质感，线条为深褐色，边缘柔和，
用于游戏UI/图标/场景（无任何文字、无水印、无logo、无字母、无数字）。
```

❌ 禁止词（容易把风格带偏）：
- cyberpunk / neon / realistic / 3D render / anime / glowing text / logo / watermark
