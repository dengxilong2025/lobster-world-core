## 0) P4 总目标（只抓三件事）

1) **rock 纯纹理化**：让 rock 变体不再像“场景块/铺装块/带符号贴花”，而是可以长期铺地的基础纹理。  
2) **biome 语义一致性**：文件名 biome 与画面内容一致（grass 就是草地，sand 就是沙地，rock 就是碎石/岩质纹理）。  
3) **交付规范收紧**：checks 覆盖更全面 + 打包一致，减少往返。

## 1) P4-1（必须）：rock 纯纹理化（替换问题变体）

### 1.1 为什么
当前 rock 01–18 中仍存在：
- **符号/装饰物**（会产生强重复印章感）
- **大主体/铺装块**（像路面/石板/图案块，不适合作为基础纹理）
- **内容跑偏**（更像 dirt/grass/sand）

### 1.2 需要你替换的 rock 变体（优先级从高到低）
> 这些编号可以保持不变（直接替换 PNG 内容），以减少我们侧的清单/引用变更。

**必须替换（强烈建议）：**
- rock_01：存在明显符号/装饰物
- rock_04：大主体花纹（强图案感）
- rock_07：明显草地+石头点缀（更像 grass 场景）
- rock_09 / rock_10 / rock_11：大块石板/铺装块倾向（印章感强）
- rock_16：更像 dirt+植物点缀（内容跑偏）
- rock_18：明显鹅卵石铺路倾向（铺装感强）

**建议替换（视产能）：**
- rock_15：更像 sand/卵石地（语义边界不清）

### 1.3 产出要求（硬规格）
- 256×256 PNG
- 无文字/水印/签名
- 只做“基础纹理尺度”：细碎碎石/砂砾/裂纹/噪点，不要大主体、不要边框、不要图案符号

## 2) P4-2（必须）：biome 语义一致性（grass/dirt/sand/rock）

### 2.1 做法（推荐）
对每个 biome（grass/dirt/sand/rock），请你在 README 里列出：
- **推荐入库 Top8**（你自评最像该 biome、最不重复、最不出接缝的 8 张）
- **疑似语义跑偏清单**（你自己觉得可能不太像该 biome 的编号）

我们会优先把 Top8 当成后续 production 主力集合。

### 2.2 红线
出现以下任意一条，都算该变体“不建议入库”：
- 画面语义明显跑偏（grass 像雪地、sand 像岩石板、rock 像草地等）
- 有明显符号/贴花/装饰物
- 有明显边框/分割线（铺开会露馅）

    ...
