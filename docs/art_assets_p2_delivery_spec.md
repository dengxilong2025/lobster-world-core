## 0. P2 你要做的事（先看这一段）

P2 优先级从高到低：

1) **重做 Type A 基础地表瓦片（必须）**：grass / dirt / sand / rock  
   - 每类 **至少 6 张变体**（01..06）  
   - 目标是“纹理瓦片”，不要大主体、不要边缘装饰
2) **补齐 Type C 过渡瓦片（强烈建议）**：草→土、土→沙、土→石、沙→石  
   - 每个过渡至少 **4 张变体**（01..04）
3) **可选加分**：snow / shallow_water 的 Type A（各 4~6 张）
4) **如果你想做路面/石板路**：请作为“路径类（path/road）”单独交付，不要放进 rock 的 Type A 里

## 1. 风格要求（沿用 P1）

关键词：
- 石器时代、部落感、暖色、手绘绘本、自然材质（泥土/砂砾/碎石）

红线：
- 任何文字/字母/水印/签名
- 赛博霓虹、冷色主导
- 明显 AI 伪影（竖向拉丝、结构断裂、异常色块）

## 2. 统一规格（必须严格满足）

### 2.1 Type A / Type C 尺寸与格式
- **256×256**
- PNG（RGB 即可；如果做 alpha，也必须合理，不允许大面积误透明）

### 2.2 命名规则

**Type A：**
```
tile_{biome}_basic_{variant}_w256_h256.png
```
biome：`grass | dirt | sand | rock | snow | water_shallow`

例：
- `tile_grass_basic_01_w256_h256.png`

**Type C（过渡）：**
```
tile_{from}_to_{to}_transition_{variant}_w256_h256.png
```
例：
- `tile_grass_to_dirt_transition_01_w256_h256.png`

**路径类（可选）：**
```
tile_path_stone_{variant}_w256_h256.png
tile_path_dirt_{variant}_w256_h256.png
```

## 3. P2 必做：Type A “真正基础纹理瓦片”（每类 6 张起）

### 3.1 本次最重要的要求：内容尺度
Type A 必须像“纹理”，而不是“场景块”：

✅ 允许出现（小尺度）：
- 细碎砂砾点
- 很小的碎石子（不要一整圈大石头沿边）
- 很细的裂纹/泥点（不要形成大块“拼图”边界）
- 非常稀疏的小草叶点（像噪点一样分布）

❌ 禁止出现（大主体/边缘装饰）：
- 草丛沿边一圈、石头沿边一圈、明显的“边框”
- 大石块、大木桩、骨头、灌木团
- 明显的板块分割（像石板路/拼装地砖）
- 中心区域出现“方形框/边界痕迹”

一句话：**铺 10×10 的时候，你看不出“每块瓦片的边界”，也看不出“同一张图在重复”。**

### 3.2 冗余交付（我鼓励你多做）
每类至少 6 张，但你可以一次交 8~12 张，我会筛选最合格的入库：
- grass：01..08（建议）
- dirt：01..08（建议）
- sand：01..08（建议）
- rock：01..08（建议）

## 4. P2 强烈建议：Type C 过渡瓦片（每对 4 张起）

### 4.1 过渡组合（优先级）
1) grass → dirt（01..04）
2) dirt → sand（01..04）
3) dirt → rock（01..04）
4) sand → rock（01..04）

### 4.2 过渡表现方式建议
- 过渡边缘要“自然侵蚀/混合”，不要硬切线
- 不要做成“装饰边框带”
- 过渡宽度建议占画面 20%~40%，别太窄（容易看出重复边）

## 6. 自检与验收方式（你交付前自己先做）

### 6.1 3×3 拼贴自检（必须）
你交付时请继续附带 `checks/3x3_*.png`（可选，但强烈建议）。  
自检时重点看：
- 接缝处有没有竖线/横线
- 10×10 铺图时有没有明显“重复印章感”

### 6.2 我这边的验收
我会用 `/ui/assets` 的工具做：
- `tiles.base` 的 3×3 拼贴预览
- alpha 异常提示（防止误透明/glitch）
- 一键导出拼贴 PNG + 留档（方便反馈与你迭代）
